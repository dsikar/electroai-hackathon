# Specification: Falstad Screenshot Automation

## 1. Overview
This specification defines the requirements and implementation for an automated pipeline to generate high-quality screenshots of Falstad electronic circuits. The process must be fully automated, headless, and integrated as a post-processing step for the schematic analysis pipeline.

## 2. Input Requirements
For each circuit, the tool must process the following inputs generated by the primary pipeline:
- **Falstad Plain-Text Netlist:** A text file starting with `$`, as specified in `prompts/04-output-spec-prompt.md` (e.g., `<stem>_generated-by-<agent>_falstad.txt`).

## 3. Output Requirements
The tool must produce exactly **two image files** in `simulations/<agent>/`:

| File | Description |
|------|-------------|
| `<stem>_generated-by-<agent>_falstad.png` | A high-DPI screenshot of the circuit canvas. |
| `<stem>_generated-by-<agent>_falstad_full.png` | A full-viewport screenshot including the UI and scopes. |

## 4. Technical Implementation (Playwright)
The automation must use **Playwright** (Python or Node.js) to interact with the Falstad/CircuitJS web application.

### 4.1 URL Construction
The circuit must be loaded via the `cct` URL parameter to avoid manual "Open File" interactions:
`https://www.falstad.com/circuit/circuitjs.html?hideSidebar=true&cct=<URL_ENCODED_NETLIST>`

### 4.2 Browser Configuration
- **Engine:** Chromium.
- **Mode:** Headless.
- **Viewport:** Minimum 1400x900 pixels.
- **Scale Factor:** 2.0 (for high-DPI/retina quality).

### 4.3 Rendering Logic
1.  **Load Page:** Navigate to the constructed URL.
2.  **Wait for Idle:** Wait for `networkidle` or a minimum of 2500ms to ensure the simulation engine has parsed the netlist and rendered the components.
3.  **Capture Canvas:** Locate the `<canvas>` element to capture the clean circuit diagram.
4.  **Capture Viewport:** Capture the full page for the UI/Scope context.

## 5. Integration
The screenshot utility must be executable as a standalone script or a module within the `scripts/` directory, taking the `<agent>` and `<stem>` as arguments to locate files in the `simulations/` hierarchy.

## 6. Error Handling
- **Timeouts:** If the page fails to load within 15 seconds, log an error and skip the circuit.
- **Missing Inputs:** Verify the existence of the `.txt` netlist before launching the browser.
- **Empty Canvas:** Detect if the canvas is empty (e.g., failed import) and log a warning.
