# Specification: LTSpice Screenshot Automation (Linux + Wine)

## 1. Overview
This specification defines the requirements and implementation for an automated pipeline to generate high-quality screenshots of LTSpice schematic files (`.asc`). Since LTSpice lacks a native command-line image export, this process uses a combination of Wine, `xdotool`, and ImageMagick's `import` utility.

## 2. Input Requirements
For each circuit, the tool must process the following input generated by the primary pipeline:
- **LTSpice Schematic File:** A `.asc` file as specified in `prompts/04-output-spec-prompt.md` (e.g., `simulations/<agent>/<stem>_generated-by-<agent>.asc`).

## 3. Output Requirements
The tool must produce exactly **one image file** in `simulations/<agent>/`:

| File | Description |
|------|-------------|
| `<stem>_generated-by-<agent>_ltspice.png` | A high-quality screenshot of the LTSpice schematic window. |

## 4. Technical Implementation (Bash/Wine/X11)
The automation must orchestrate the LTSpice GUI environment on a Linux host (or within a container with Xvfb).

### 4.1 Execution Environment
- **Wine:** Must be configured to run the target LTSpice executable (e.g., `XVIIx64.exe`).
- **X11 Utilities:** `xdotool` for window manipulation and `import` (ImageMagick) for screen capture.

### 4.2 Automation Logic
1.  **Launch:** Execute `wine <LTSPICE_EXE> <ASC_FILE>` as a background process.
2.  **Window Detection:** Poll the window manager using `xdotool search --name "LTspice"` until the schematic window is visible (timeout: 15 seconds).
3.  **Preparation:**
    - Focus and activate the window.
    - Maximize the window to ensure high resolution.
    - Send the `Space` key to the window to trigger the "Zoom to Fit" command.
    - Wait for rendering (minimum 1000ms).
4.  **Capture:** Use `import -window <WINID>` to capture the specific LTSpice window decoration and canvas.
5.  **Termination:** Close the LTSpice process gracefully after capture.

## 5. Integration
The screenshot utility must be executable as a standalone Bash or Python script within the `scripts/` directory, accepting the `<agent>` and `<stem>` as arguments to locate files in the `simulations/` hierarchy.

## 6. Alternative Implementations
- **Headless (CI/CD):** If running on a server without a physical display, the script must be executed via `xvfb-run`.
- **Vector Alternative:** For publication-quality diagrams without GUI dependencies, the use of `ltspice2svg` followed by `rsvg-convert` is permitted as an alternative path.

## 7. Error Handling
- **Wine Failures:** Log an error if the LTSpice executable is not found or fails to launch under Wine.
- **Window Timeout:** Kill the process if the window does not appear within the 15-second polling window.
- **Missing File:** Verify the existence of the `.asc` file before execution.
